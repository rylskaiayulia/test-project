REST API запрос

Сценарий работы
1. Пользователь открывает экран «Выберите магазин».
2. Приложение вызывает:

GET /api/v1/partner-stores

3. Сервер возвращает JSON со списком магазинов.
4. Приложение отображает карточки.
5. При нажатии на карточку магазина идет перенаправление/редирект на redirectUrl.

Метод:

GET

Endpoint:

/api/v1/partner-stores

Пример запроса

GET /api/v1/partner-stores HTTP/1.1
Host: api.petrushka-green.ru
Authorization: Bearer <token>
Accept: application/json

Описание компонентов
Cart Service / Order Service / Marketing Service
Создают события.
Примеры:
* корзина долго не изменялась;
* заказ отменён;
* стартовала рекламная рассылка.

Message Broker
Посредник между сервисами.
Функции:
* принимает события;
* временно хранит их;
* передаёт подписчикам.
Пример события:
{
  "eventType": "CART_ABANDONED",
  "userId": 123,
  "cartId": 555
}

Notification Service
Подписывается на события брокера.
Функции:
* получает событие;
* определяет шаблон уведомления;
* получает push-токен пользователя;
* отправляет PUSH.

Пример сформированного уведомления:
{
  "title": "Корзина ждёт вас",
  "body": "Завершите оформление заказа"
}

Push Provider
Сервис доставки уведомлений:
* Android → FCM
* iOS → APNs

Сценарий работы
1. Пользователь оставил товары в корзине.
2. Cart Service обнаружил отсутствие активности.
3. Cart Service публикует событие в Message Broker.
4. Notification Service получает событие.
5. Формирует PUSH.
6. Отправляет уведомление через Push Provider.
7. Пользователь получает PUSH.
